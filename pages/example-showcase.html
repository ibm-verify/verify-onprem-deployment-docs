# Comprehensive Schema Example

This schema demonstrates all the capabilities supported by the generate_schema_docs.py script.
It includes examples of:

- Multiple JSON Schema versions (Draft-04 through 2020-12)
- Complex nested objects and arrays
- All validation constraints
- References ($ref) to definitions
- Modern features like prefixItems, dependentSchemas, unevaluatedProperties
- Markdown formatting in descriptions
- Examples and default values
- Enum and const values

## Usage

This schema can be used to validate configuration files for a fictional **Cloud Application Platform**.

ℹ️ Click on section headers below to expand and view configuration details.

Configuration

▼

application object required

▼

The main application configuration section.

This section defines core application settings including:
- Application metadata (name, version)
- Runtime environment settings
- Feature flags

⚙️ Properties
name string required

The unique name of the application

Validation Constraints:
• Min length: 3
• Max length: 63
• Pattern: ^[a-z][a-z0-9-]*$
version string required

Semantic version of the application.

Must follow [SemVer 2.0.0](https://semver.org/) format.

Validation Constraints:
• Pattern: ^\d+\.\d+\.\d+(-[a-zA-Z0-9.-]+)?(\+[a-zA-Z0-9.-]+)?$
environment string

The deployment environment

Default: development
Allowed values:
development staging production
debug boolean

Enable debug mode with verbose logging

Default: False

Examples

▼
Example 1
application:
  name: my-cloud-app
  version: 1.0.0
  environment: production
  debug: false
  features:
    analytics: true
    darkMode: true
    experimentalUI: false
server:
  host: 0.0.0.0
  port: 8080
  protocol: https
  ssl:
    enabled: true
    certificate: /etc/ssl/certs/server.crt
    privateKey: /etc/ssl/private/server.key
    protocols:
    - TLSv1.3
  timeout: 30000
  maxConnections: 1000
database:
  type: postgresql
  connection:
    host: db.example.com
    port: 5432
    database: myapp
    username: dbuser
    password: encrypted_password
  pool:
    min: 2
    max: 10
    idleTimeout: 10000
logging:
  level: info
  format: json
  outputs:
  - type: console
  - type: file
    path: /var/log/app.log
    maxSize: 100MB
security:
  authentication:
    enabled: true
    providers:
    - oauth2
    - local
  cors:
    enabled: true
    origins:
    - https://example.com
    methods:
    - GET
    - POST
    - PUT
    - DELETE
    credentials: true
  rateLimit:
    enabled: true
    requests: 1000
    window: 60
    strategy: sliding-window
monitoring:
  metrics:
    enabled: true
    endpoint: /metrics
    interval: 60
  healthCheck:
    enabled: true
    endpoint: /health
    checks:
    - database
    - cache
  tracing:
    enabled: true
    provider: jaeger
    samplingRate: 0.1
deployment:
  replicas: 3
  strategy: rolling
  resources:
    limits:
      cpu: '2'
      memory: 2Gi
    requests:
      cpu: 500m
      memory: 512Mi
cache:
  enabled: true
  provider: redis
  ttl: 3600
  maxSize: 1GB
